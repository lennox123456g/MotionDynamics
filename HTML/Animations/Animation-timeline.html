The Art of Motion

Animation on the web has come a long way. What once required heavy JavaScript libraries can now be achieved with a few lines of CSS — smooth, performant, and tied directly to the browser's rendering engine.

Scroll-driven animations

The animation-timeline: scroll property lets you bind any CSS animation to a scroll position instead of time. As the user scrolls down, the animation progresses — no JavaScript event listeners needed.

Why it matters

Traditional scroll animations fired JS callbacks on every scroll event, causing layout thrashing and janky framerates. Native scroll timelines run off the main thread, meaning silky 60fps even on low-end devices.

The progress bar trick

That pink bar at the top of this page is a fixed element scaled from 0 to 1 on the X axis, driven entirely by how far you've scrolled. No scroll listeners. No requestAnimationFrame. Just CSS.

What comes next

Beyond scroll(), the spec also defines view() — a timeline that activates when an element enters or leaves the viewport. Think reveal animations, sticky counters, and parallax effects, all in CSS.
